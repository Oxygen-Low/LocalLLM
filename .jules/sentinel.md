## 2024-05-18 - Missing SSRF Validation in SEO Apps Endpoint
**Vulnerability:** The `POST /api/web-seo/apps` endpoint accepted user-provided URLs (`url` for URL apps and `cloneUrl` for Repo apps) without validating them against Server-Side Request Forgery (SSRF). This allowed users to point the SEO analysis tool (which runs Playwright in a Docker container) at internal or private IP addresses (e.g. `127.0.0.1`, `169.254.169.254`).
**Learning:** Even if a process runs inside a Docker container, it is still susceptible to SSRF if the container shares network access with the host or internal resources (such as via a bridge network or `host.docker.internal`). User-supplied URLs must be validated before the server attempts to access them or pass them to another internal process for fetching.
**Prevention:** Always use established SSRF-safe URL validation functions (like the existing `ssrfSafeUrlValidation` in `server/server.js`) on all user-supplied URLs before using them. Ensure validation covers both the scheme (HTTP/HTTPS only) and the resolved IP addresses (rejecting private/reserved IP ranges).
## 2024-04-02 - Command Injection in Docker Exec via Shell Interpolation
**Vulnerability:** The endpoints `GET /api/coding-agent/containers/:id/files` and `POST /api/coding-agent/containers` constructed `bash -c` command strings by directly interpolating user inputs (`dirPath`, `branchName`, and `cloneUrl`). This allowed users to inject arbitrary shell commands (e.g., `"; touch pwned; "`) to be executed within the context of the running Docker container.
**Learning:** Even when using array-based arguments for child processes (like `execFileSync('docker', ['exec', ..., 'bash', '-c', "..."])`), passing unsanitized user inputs into a shell's `-c` evaluation string exposes the system to command and argument injection.
**Prevention:** Always base64-encode user inputs within Node.js before passing them to a shell context, then dynamically decode them inside the script (e.g., `find "$(echo '${b64DirPath}' | base64 -d)"`) to eliminate interpolation risks while bypassing quoting and word-splitting issues.
